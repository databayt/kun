## Build commands

```bash
pnpm install --frozen-lockfile     # install deps
pnpm dev                            # start docs site on :3000 (auto-Chrome via hook)
pnpm build                          # production build
pnpm typecheck                      # TS check (no emit)
pnpm lint                           # ESLint
pnpm test                           # Vitest run
pnpm test --watch                   # Vitest watch
```

Engine commands (operate on `.claude/` itself):

```bash
bash scripts/inventory.sh                            # → docs/INVENTORY.md
bash scripts/runway.sh                                # → .claude/memory/runway.json
bash scripts/migrate-commands-to-skills.sh [--force] # commands/ → skills/
bash scripts/build-plugins.sh [name|all]              # → plugins/<name>/
bash scripts/setup-routines.sh [--list|--show <id>]   # routine registration help
bash scripts/setup-apple-notes.sh                     # Mac: dispatch channels
bash scripts/session-start.sh                          # SessionStart hook (auto)
```

## Fast local development

For doc-site changes:
```bash
pnpm dev    # auto kills :3000, opens Chrome
```

For engine changes (no rebuild needed — files are read at session start):
```bash
# Edit .claude/agents/* or .claude/skills/*
# Open a fresh Claude Code session — changes are live
# Or use /reload-plugins if working in a plugin
```

For hook changes:
```bash
# Edit scripts/hooks/*.sh or .claude/settings.json
# Open a fresh session — hooks are bound at startup
```

## Testing

```bash
pnpm test                          # all unit tests
pnpm test path/to/file.test.tsx    # one file
pnpm test --coverage               # coverage report
```

E2E (when needed for full-stack admission flow on hogwarts):
```bash
pnpm test:e2e                       # Playwright
```

For the engine itself, smoke tests are run by `scripts/inventory.sh` (filesystem checks) and the Routines `daily-org-pulse` (live state).

### Writing tests

Follow `.claude/rules/testing.md`. Highlights:

- **Do**: hit a real database for tenant-owned actions; use `userEvent` not `fireEvent`; one assertion per logical concern
- **Don't**: mock the database in multi-tenant tests; snapshot tests for DOM; `it.skip` without an issue link

## Linting and types

```bash
pnpm typecheck && pnpm lint
```

CI fails on:
- `any` (per `.claude/rules/typescript.md` — but kun docs site does not yet have this rule active)
- `// @ts-ignore` without a linked issue
- ESLint errors (warnings allowed)

Fix loop:
```bash
/fix              # auto-resolves trivial lint/type errors
/typescript        # full coverage sweep across src/
```

## PR status (CI failures and reviews)

```bash
gh pr checks                       # CI status for current PR
gh pr view --comments              # review comments
gh run watch                       # watch CI live
gh api repos/databayt/kun/pulls/{n}/reviews
```

If a CI failure looks flaky, re-run the failed job:
```bash
gh run rerun <run-id> --failed
```

Do **not** force-push to PRs under review — captain will block via `runway-critical` rule if the PR is on a critical-path branch.

## PR descriptions

Use the GitHub PR template (`.github/PULL_REQUEST_TEMPLATE.md`). For ad-hoc PRs:

```
## Summary
<1-3 bullets>

## Changes
- <file or area>: <what>
- <file or area>: <what>

## Test plan
- [ ] <verification step>

Closes #<issue-number>
```

Closes #N auto-closes the issue. Always link the issue.

Do **not** add "Generated with Claude Code" co-author lines to PR bodies. Co-authorship goes in the commit trailer (per `.claude/rules/github-workflow.md`).

## Development tips

- **Open a fresh session after editing `.claude/`** — most changes need a session restart to take effect
- **`/reload-plugins`** picks up plugin changes without a restart
- **`bash scripts/inventory.sh`** before opening a PR that touches `.claude/` — catches drift
- **Cost** — use `claude-haiku-4-5` for read-only research; `claude-opus-4-7` only for strategic work. See `.claude/cost/routing.yaml`.

## Secrets and env safety

- ONE `.env` file per repo, at the repo root. Never commit. Never create `.env.local`/`.env.development`/etc.
- Secrets via macOS Keychain. Setup: `bash ~/.claude/scripts/setup-secrets.sh`
- Vercel env vars mirror `.env`. Never set per-deploy overrides via dashboard.
- API keys (Anthropic, Stripe, GitHub PAT) live only in Keychain + Vercel env. Never in code, never in commits, never in chat.

## Specialized skills

Heavy/conditional workflows live in `.claude/skills/<name>/SKILL.md`:

- `/captain` — captain decision loop (`--dry-run` to preview)
- `/sprint-plan` — Monday allocation
- `/standup` — daily digest
- `/sprint-review` — Friday retro
- `/refine` — backlog grooming with ICE
- `/dispatch` — atomic write to Apple Notes + Slack + bridge.md
- `/weekly` — captain weekly cycle (orchestrates `/sprint-plan` + dispatches)
- `/monitor` — cross-product health check
- `/costs` — Anthropic + Vercel + Neon + Stripe spend
- `/deploy`, `/ship` — production deploy (manual-only, `disable-model-invocation`)
- `/feature` — full pipeline idea → spec → schema → code → wire → check → ship → watch
- `/report` — auto-fix user-reported bugs
- 14 sweep skills (`/nextjs`, `/react`, `/typescript`, `/tailwind`, `/shadcn`, `/prisma`, `/authjs`, `/accessibility`, `/barrel`, `/waterfall`, `/skeleton`, `/structure`, `/guard`, `/translate`)

Skills are path-scoped where applicable — they auto-load only when relevant files are open. See each `SKILL.md`'s `paths:` frontmatter.

## Context-efficient workflows

Token budgets matter. The kun engine's design optimizes for them:

- **Path-scoped rules** (`.claude/rules/<topic>.md` with `paths:` frontmatter) load only when relevant files are open
- **Sweep skills** (`/nextjs`, `/react`, etc.) batch by module and use a per-product ledger to skip already-checked code
- **Captain `--dry-run`** previews decisions without writing to memory
- **`context: fork`** on heavy ceremonies (`/weekly`, `/dispatch`, `/captain`) keeps the main session clean
- **Auto-memory at `~/.claude/projects/-Users-abdout-kun/memory/`** — Claude writes own notes, indexed by `MEMORY.md`
- **PreCompact hook** snapshots captain state before context compression

Don't re-read the same file in a session. Use `Read` once, then refer to the line numbers.

## Commit and PR style

- **Conventional Commits**: `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`, `test:`, `perf:`, `style:`
- Subject ≤ 72 chars, present tense ("add" not "added")
- Body explains WHY (the diff shows WHAT)
- One concern per commit; one feature per PR
- Reference the issue in the trailer: `Refs #N` or `Closes #N`
- Co-author trailer for AI: `Co-Authored-By: Claude Opus 4.7 <noreply@anthropic.com>`

PRs:
- Squash merge to `main`
- Delete branch after merge
- Never `--no-verify`, never force-push to `main`
- Captain auto-checks every PR via `report-auto-fix` Routine if labeled `report`

## Task decomposition and verification

Break work into stories that fit a single Claude Code session with full context (BMAD discipline). Each story:
1. Names exact files
2. Names exact frontmatter fields / API shapes
3. Names exact verification (a command + expected output, OR a `/watch` URL)

Verify each story before moving to the next. Don't bundle multiple stories into one PR unless they share a single concern.

## Rebuilding before running tests

If you've edited `.claude/agents/<n>.md`, `.claude/skills/<n>/SKILL.md`, or `.claude/settings.json`:
1. Open a fresh session (or run `/reload-plugins`)
2. Verify the change is loaded: `grep "..." ~/.claude/.cache/...` or run `/agents` / `/skills` listing

For doc-site code in `src/`:
```bash
pnpm install      # if package.json changed
pnpm typecheck && pnpm build
```

## Anti-patterns (don't do these)

- ❌ `model: opus` literal in agent frontmatter — use `claude-opus-4-7` (full ID, version-locked)
- ❌ Hardcoded paths in skills — use `$CLAUDE_PROJECT_DIR` or relative paths
- ❌ Bash hooks with `set -e` — read-only diagnostics should never block; use `trap 'exit 0' ERR`
- ❌ `disallowedTools` missing on leadership agents — they should never `Write`/`Edit`
- ❌ Path-scoped rule without a `paths:` field — wastes context
- ❌ New skill without `disable-model-invocation: true` for destructive operations
- ❌ Captain dispatching `urgent` for non-urgent things (priority erosion)
- ❌ Routine without an estimate in `manifest.json.estimatedMonthlyCost`
- ❌ Plugin manifest missing `license` field
- ❌ Memory file edited by hand without bumping `lastUpdated`
- ❌ `--no-verify` git operations (the hook is there for a reason)

## Test gotchas

- **Auto-memory writes interfere with snapshot tests** — clear `~/.claude/projects/<project>/memory/` before snapshot runs
- **Routine cost tests** must mock the cost-watcher hook output (don't actually hit Anthropic API)
- **Dispatch tests** on Mac must `osascript` (Apple Notes) — wrap in `if [ "$(uname)" = "Darwin" ]; then`
- **Sweep skills** require a per-product ledger to be present — initialize via `/coverage` first
- **Captain `--dry-run`** must NOT update `captain_journal.md` — verify with `git diff` after running
