## Accessibility — operating notes

**Critical**: Samia is blind and uses a screen reader (likely NVDA on Windows). Operational implications:

1. **Voice-first interaction preferred.** Schedule voice/video meetings rather than text-only.
2. **All written follow-ups must be screen-reader friendly**: proper markdown headings (H1 → H2 → H3 hierarchy, no skipping), alt text on images, no ASCII tables (use markdown tables), no emoji-only navigation.
3. **Test any UI Samia will use** with NVDA or VoiceOver before delivering.
4. **When recommending tools**, prioritize ones with screen-reader support documented (e.g., GitHub web is good, Discord is mediocre).
5. **Don't ask her to "just check this screenshot"** — surface decisions in text.
