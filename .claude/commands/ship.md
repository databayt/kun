# Ship — Deploy to Production

The penultimate pipeline stage. Take a green `/check` and put it in front of customers. Chains build → quality gate → Vercel deploy → autofix-pr (web-side PR babysitter).

## Usage
- `/ship` — deploy current branch
- `/ship admission` — deploy a specific feature branch
- `/ship --pr` — push first, open PR, then deploy via PR
- `/ship --skip-autofix` — deploy without the autofix-pr web session

## Argument: $ARGUMENTS

## Instructions

### 1. PRE-FLIGHT — Verify branch state

```bash
git status
git branch --show-current
```

Rules:
- Must be on a non-`main` branch (no direct main deploys)
- Working tree must be clean (no uncommitted changes)
- Branch must be ahead of `origin/main` (something to ship)

If any rule fails: report and stop. Do NOT auto-commit or auto-push.

### 2. BUILD — Run /build

Invoke the build skill / command. Wait for green.

```bash
pnpm tsc --noEmit && pnpm build
```

If fails: stop. Delegate fixing back to `/code` or `/wire`.

### 3. CHECK — Run /check

Invoke the quality-gate command. Wait for `READY TO SHIP` verdict.

If `BLOCKED`: stop. Report which checks failed.

### 4. PUSH — Push to remote

```bash
git push -u origin "$(git branch --show-current)"
```

If branch tracks no remote: the `-u` sets it. If remote rejects (non-fast-forward): rebase first, do NOT force push without confirmation.

### 5. PR — Open or update pull request

If no PR exists for the branch:

```bash
gh pr create --title "<type>: <description>" --body "$(cat <<'EOF'
## Summary
<auto-generated from commits>

## Test plan
- [ ] CI passes
- [ ] Vercel preview deploys
- [ ] /watch confirms healthy

Closes #<N>

🤖 Generated with [Claude Code](https://claude.com/claude-code)
EOF
)"
```

If a PR exists: skip; capture the PR number for step 7.

### 6. DEPLOY — Vercel

For most kun-managed repos, push triggers Vercel automatically. Verify:

```bash
npx vercel list --yes --limit 1
```

Wait for status to reach `Ready` (poll every 30s, max 10 minutes). If `Error`: fetch logs, report, stop. If `Building`: keep polling.

### 7. AUTOFIX-PR — Hand off to web session (default)

Unless `--skip-autofix`, kick the autofix-pr babysitter that watches the PR in a Claude Code web session:

Invoke `/autofix-pr <PR-number>` — see `.claude/commands/autofix-pr.md`. The web session monitors CI, applies suggested review fixes, and keeps the PR mergeable.

### 8. HANDOFF — Report

```
## Ship Report

**Branch**: <branch>
**PR**: <PR URL>
**Deployment**: <Vercel preview URL>
**Status**: Ready
**Autofix-PR**: <web session URL or "skipped">

Next: `/watch` against the preview URL when ready.
```

## Error Recovery

| Error | Action | Allow auto-fix? |
|-------|--------|-----------------|
| On `main` branch | Stop, ask user | No |
| Uncommitted changes | Stop, ask user | No |
| Build fails | Delegate to `/code` | No (user-driven) |
| Check fails | Delegate to `/wire` or `/code` | No (user-driven) |
| Push rejected | Rebase, retry once | Yes (no force push) |
| PR exists with conflicts | Report, ask user | No |
| Vercel build error | Fetch logs, report | No (delegate) |

## Exit Gate

- Branch pushed to remote
- PR opened or updated
- Vercel deployment `Ready`
- Autofix-PR running (or explicitly skipped)
- `/watch` is the next stage in the pipeline
