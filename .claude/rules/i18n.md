# i18n Rules

Active in any localized route or component path. Arabic is the default; English is opt-in.

## Locale routing

App Router uses `[lang]` segment. Every page receives `params: { lang: "ar" | "en" }`. Dictionary loads:

```ts
import { getDictionary } from "@/dictionary";

export default async function Page({ params }: { params: Promise<{ lang: "ar" | "en" }> }) {
  const { lang } = await params;
  const dict = await getDictionary(lang);
  return <Heading>{dict.page.title}</Heading>;
}
```

`params` is a Promise in Next.js 16 — always `await` it.

## Dictionaries

`src/dictionary/{ar,en}.json` are mirror trees. Adding a key in `ar.json` requires adding the same key in `en.json` (and vice versa). The TypeScript Dictionary type derived from `ar.json` is the source of truth.

Never hardcode user-facing strings in JSX. Strings only via `dict.<section>.<key>`.

## RTL

RTL is automatic from `<html lang dir>` set in the root layout:

```tsx
<html lang={lang} dir={lang === "ar" ? "rtl" : "ltr"}>
```

Components must use **logical properties only**:

| Wrong | Right |
|-------|-------|
| `ml-4`, `mr-4` | `ms-4`, `me-4` |
| `pl-4`, `pr-4` | `ps-4`, `pe-4` |
| `text-left`, `text-right` | `text-start`, `text-end` |
| `left-0`, `right-0` | `start-0`, `end-0` |
| `border-l`, `border-r` | `border-s`, `border-e` |
| `rounded-l`, `rounded-r` | `rounded-s`, `rounded-e` |

Use `flex-row` then add `rtl:flex-row-reverse` only when icon + label visual order must flip.

## Locale-aware formatting

Numbers, dates, currencies via Intl, never via `toLocaleString` without an explicit locale:

```ts
new Intl.DateTimeFormat(lang === "ar" ? "ar-SA" : "en-US", {
  dateStyle: "medium",
}).format(date);

new Intl.NumberFormat(lang === "ar" ? "ar-SA" : "en-US").format(price);
```

Saudi calendar (Hijri) optional — only when `dict.calendar.system === "hijri"`.

## Arabic-first content

When writing default copy, write in Arabic first. The English version is a translation, not the source. Reflects Saudi-first market.

## Never

- Use `text-left` / `text-right` — breaks RTL
- Hardcode strings in JSX outside of `lib/constants/` (which is non-translatable)
- Skip the en mirror when adding an ar key
- Use a 3rd-party i18n library — Anthropic's dictionary pattern is the convention
- Conditional render based on `lang` if a token solves it (`dir="rtl"` + logical property is preferred)
