# Plan template

The canonical shape of a `/plan` issue comment. Posted as a comment headed `## Implementation Plan`.

```markdown
## Implementation Plan

### Pattern Choices

- Form: <pattern card, e.g. patterns/cards/form.md — central-block>
- Table: <patterns/cards/table.md — content+table+columns triplet>
- Rules in force: <.claude/rules/<domain>/ files that govern this feature>

### Architecture Decisions

- Data flow: <server component fetches → form posts to server action>
- Boundaries: <what is server vs 'use client'; minimize client>
- Caching: <'use cache' on which reads; revalidateTag/updateTag on writes>
- Auth + tenant: <guard at layout/action; tenant-isolation field threaded>

### Mirror Target

<the existing feature this one copies, with path — e.g. src/components/hogwarts/payments/>

### Tradeoffs

<the 1-2 real forks in the road and why this side>

### Risks

<migrations touching live tenants; breaking changes; perf hotspots>

---

_Generated by plan stage. Ready for task breakdown._
```

Owned by `tech-lead` + the user-level `architecture` agent. Cite pattern cards and rule files by name so `/check` and `/handover` can hold the implementation to them.
