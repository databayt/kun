# Spec template

The canonical shape of a `/spec` issue comment. Posted as a comment headed `## Technical Spec`.

```markdown
## Technical Spec

### Data Model

<prisma sketch — fields, relations, tenant-isolation field>

### File Plan

Files to create:
src/components/{scope}/{name}/content.tsx — server component, data fetching
src/components/{scope}/{name}/actions.ts — server actions (CRUD)
src/components/{scope}/{name}/form.tsx — client form component
src/components/{scope}/{name}/columns.tsx — table columns (if list view)
src/components/{scope}/{name}/validation.ts — Zod schemas
src/app/[lang]/{route}/{name}/page.tsx — page (mirror pattern)
Files to modify:
prisma/schema.prisma (or prisma/models/) — add model
src/dictionaries/{en,ar}.json — i18n keys

### Acceptance Criteria (Refined)

<required vs optional fields; which actions need auth guards; which views;
RTL/LTR considerations>

### Dependencies

<external packages; MCP tools; existing components to reuse>

---

_Generated by spec pipeline. Ready for plan._
```

The `/spec` stage ends with the **one human checkpoint** in the pipeline. Everything after (plan, tasks, schema…) is automated.
