# Tasks template

The canonical shape of a `/tasks` issue comment. Posted as a comment headed `## Task Breakdown`, rendered as a GitHub checklist grouped by pipeline stage so progress is trackable on the issue.

```markdown
## Task Breakdown

### schema

- [ ] Add `<Model>` to prisma/schema.prisma with tenant field — verify: `pnpm tsc --noEmit`
- [ ] Create validation.ts Zod schemas — verify: types compile

### code

- [ ] Server actions (create/update/delete) with auth + tenant scope — verify: `pnpm tsc --noEmit`
- [ ] Authorization rules — verify: unauthorized call rejected

### wire

- [ ] page.tsx (mirror pattern) + content.tsx — verify: route renders
- [ ] form.tsx with useActionState — verify: submit round-trips
- [ ] columns.tsx (if list view) — verify: table renders
- [ ] i18n keys (en + ar) — verify: no hardcoded strings — `pnpm build`

### check

- [ ] /handover <block> — verify: READY FOR DEMO

---

_Generated by tasks stage. Ready for implementation._
```

Order matters: dependencies first (schema → code → wire). Each task carries its own verify step (the stage's exit gate).
