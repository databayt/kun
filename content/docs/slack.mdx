# Slack

Team communication hub. Connected to Claude Code via MCP for notifications, dispatch, and collaboration.

## Channels

| Channel | Purpose |
|---------|---------|
| `#general` | Team-wide announcements |
| `#dev` | Development discussion, PR notifications |
| `#ops` | Deployments, incidents, monitoring |
| `#product` | Product-specific channel |
| `#dispatch` | Captain dispatches, weekly reviews |
| `#alerts` | Automated alerts (deploys, errors, costs) |

## MCP Integration

Slack MCP is configured in `.claude/mcp.json`:

```json
"slack": {
  "type": "http",
  "url": "https://mcp.slack.com",
  "env": {
    "SLACK_BOT_TOKEN": "${SLACK_BOT_TOKEN}",
    "SLACK_WORKSPACE_ID": "${SLACK_WORKSPACE_ID}"
  }
}
```

### Setup Steps

#### 1. Create Slack App

1. Go to [api.slack.com/apps](https://api.slack.com/apps)
2. Click **Create New App** → **From scratch**
3. Name it (e.g. `Kun Bot`)
4. Pick the workspace

#### 2. Bot Scopes

Under **OAuth & Permissions** → **Bot Token Scopes**, add:

| Scope | Purpose |
|-------|---------|
| `channels:history` | Read channel messages |
| `channels:read` | List channels |
| `chat:write` | Send messages |
| `groups:history` | Read private channel messages |
| `im:history` / `im:read` / `im:write` | DM access |
| `users:read` | List workspace members |
| `reactions:read` / `reactions:write` | Reactions |
| `files:read` | Access shared files |

#### 3. Install to Workspace

Copy the **Bot User OAuth Token** (starts with `xoxb-`).

#### 4. Get Workspace ID

```bash
curl -s -H "Authorization: Bearer xoxb-YOUR-TOKEN" \
  https://slack.com/api/auth.test | jq '.team_id'
```

#### 5. Set Environment Variables

```bash
SLACK_BOT_TOKEN=xoxb-your-bot-token
SLACK_WORKSPACE_ID=T_your_workspace_id
```

#### 6. Store in Keychain

```bash
security add-generic-password -a "slack" -s "SLACK_BOT_TOKEN" -w "xoxb-your-token" -U
security add-generic-password -a "slack" -s "SLACK_WORKSPACE_ID" -w "T_your_id" -U
```

## Usage

| Say | Action |
|-----|--------|
| `slack` | Connect to Slack MCP |
| `notify #dev "deployed v2"` | Post to `#dev` |
| `slack channels` | List all channels |
| `dispatch slack` | Send captain dispatch to `#dispatch` |

### From Agents

Agents like `captain`, `ops`, and `support` use Slack for:

- **Captain** — weekly dispatches to `#dispatch`
- **Ops** — deploy notifications, incident alerts
- **Support** — customer issue updates
- **Growth** — content announcements

## Cowork Bridge

| Channel | Cowork | Claude Code |
|---------|--------|-------------|
| `#dispatch` | Receives strategic updates | Sends build/deploy updates |
| `#dev` | Reviews PR discussions | Posts PR links, build status |
| `#alerts` | Monitors for issues | Sends automated alerts |

## GitHub Integration

Bridges Slack and GitHub — describe issues in Slack, kun creates them in GitHub.

```
Team member in Slack → describes bug/feature
       ↓
Claude Code reads Slack (Slack MCP)
       ↓
Creates GitHub issue (GitHub MCP)
       ↓
Posts issue link back to Slack
```

| You Say | Action |
|---------|--------|
| `issue from slack` | Read recent Slack messages, create GitHub issues |
| `issue "title" --repo <repo>` | Create issue, post link to `#dev` |

### Channel → Repo Mapping

| Channel | Default Repo |
|---------|--------------|
| `#<product>` | The matching product repo |
| `#dev` | Current working repo |
| `#ops` | Engine repo |
| `#alerts` | Repo from error context |

### Two-Way Sync

| GitHub Event | Channel | Message |
|--------------|---------|---------|
| Issue created | `#dev` | "New issue: #15 …" |
| PR merged | `#dev` | "Merged: #42 …" |
| Deploy succeeded | `#ops` | "Deployed v1.2.3 to production" |
| Deploy failed | `#alerts` | "Deploy failed: build error" |

## Security Notes

- Bot tokens start with `xoxb-` — never commit to git
- Store in macOS Keychain or `.env` (gitignored)
- Rotate tokens on suspected compromise
- Limit bot scopes to what's actually needed
- Bot only accesses channels it's been invited to
