# Context

A cloud tag UI where any team member clicks a keyword and dispatches a workflow. The cloud adapts to who's viewing and what stories are active.

**Route:** `/[lang]/context` (en + ar)

## How It Works

```
Team member visits /context
  → Selects their role — cloud re-weights for that role
  → Sees active stories as chips
  → Clicks a story — its keywords grow in the cloud
  → Clicks a keyword
  → Modal: confirm repo + story context
  → Submit
  → Server action creates GitHub Issue (label: dispatch)
  → Claude Code picks it up → executes → closes issue
```

## Cloud Tag

Keywords render as a tag cloud with varying sizes and weights:

- **Story boost** (+2): keyword in selected story's `keywords[]`
- **Role boost** (+1): keyword matches the viewer's role
- **Active boost** (+1): keyword in any active story
- **Base** (2): every keyword starts here

## Roles

| Role | Boosted Keywords |
|------|-----------------|
| Engineer | feature, spec, schema, code, wire, deploy, build, fix, performance |
| QA | report, check, review, test, deploy, monitor, handover, security |
| Research | translate, docs, clone, review, constitution |
| Operations | monitor, deploy, incident |

## Stories (Kanban Board)

```typescript
interface Story {
  id: string
  name: string
  nameAr: string
  repo: string
  issue?: number
  keywords: string[]
  status: "active" | "next" | "done"
}
```

## Keyword Groups

| Group | Keywords |
|-------|----------|
| **Pipeline** | feature, idea, spec, schema, ready, code, wire, check, ship, watch |
| **Tools** | dev, build, deploy, report, test, clone, monitor, atom, block, template, translate, incident |
| **Quality** | security, performance, fix, review, analyze, constitution |
| **Services** | github, vercel, neon, stripe, figma, sentry |

## Dispatch Mechanism

Clicking a keyword calls the `dispatch` server action — creates a GitHub Issue with `dispatch` label. Claude Code picks it up via remote trigger or scheduled check.

Requires `GITHUB_PERSONAL_ACCESS_TOKEN` in `.env` with `repo` scope.
