# Cowork

Cowork and Claude Code are two postures of the same brain. They share everything in `~/.claude/` — agents, memory, settings, rules.

## Two Postures

| Mode | Posture |
|------|---------|
| **Cowork** | Think. Plan. Research. Write. Decide. |
| **Claude Code** | Do. Build. Deploy. Fix. Ship. |

## Shared State

| Resource | Path | Purpose |
|----------|------|---------|
| Agents | `~/.claude/agents/` | Same agent fleet in both modes |
| Memory | `~/.claude/memory/` | Persistent state |
| Dispatch/Cowork | Notes → Dispatch → Cowork | Handoff note |
| GitHub Issues | Repo issues | Shared work queue |

## Handoff: Cowork → Code

1. Cowork produces a deliverable: plan, architecture, stories
2. Cowork creates GitHub issues for actionable items
3. Cowork writes to the Dispatch/Cowork note
4. Code reads the note at session start
5. Code picks up the issues and executes

## Handoff: Code → Cowork

1. Code writes to Dispatch/Cowork
2. Code creates a follow-up GitHub issue if needed
3. Cowork picks up in the next Desktop session

## Use Cases

### Business
```
> "Summarize this week's progress for pitch materials"
> "Draft proposal for prospective client X"
> "Research adjacent markets and competitors"
```

### Content / Research
```
> "Translate these UI strings to Arabic"
> "Write help documentation for the admission module"
> "Research best practices for school notification systems"
```

### Engineering Planning
```
> "Plan the notification system architecture"
> "Review the weekly revenue report"
> "Design the multi-tenant billing flow"
```

## Session Start Protocol

### Cowork session
1. Check Dispatch/Cowork note for Code's output
2. Review GitHub issues for completed/blocked items
3. Plan next moves

### Code session
1. `dispatch.sh read inbox`
2. `dispatch.sh read cowork`
3. `gh issue list --state open`
4. Proceed with highest priority

## The Insight

Cowork doesn't need to know how to code. Claude Code doesn't need to strategize. They share a brain through files, notes, and issues.
