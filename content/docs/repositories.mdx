Complete reference of all repositories in the databayt GitHub organization.

## Core Libraries

### codebase

Component library and patterns - the source of truth.

- **URL**: [github.com/databayt/codebase](https://github.com/databayt/codebase)
- **Stack**: Next.js 16, React 19, TypeScript, Tailwind CSS 4, Prisma 6
- **Use For**: Patterns, agents, templates, blocks, registry

```bash
# Clone
git clone https://github.com/databayt/codebase.git

# Reference pattern
# Say: "table from codebase" or "auth from codebase"
```

### shadcn

Fork of shadcn/ui with customizations.

- **URL**: [github.com/databayt/shadcn](https://github.com/databayt/shadcn)
- **Upstream**: [github.com/shadcn-ui/ui](https://github.com/shadcn-ui/ui)
- **Use For**: UI components, registry system

```bash
# Reference component
# Say: "button from shadcn" or "card from shadcn"
```

### radix

Fork of Radix UI primitives.

- **URL**: [github.com/databayt/radix](https://github.com/databayt/radix)
- **Upstream**: [github.com/radix-ui/primitives](https://github.com/radix-ui/primitives)
- **Use For**: Low-level accessible UI primitives

```bash
# Reference primitive
# Say: "dialog from radix" or "popover from radix"
```

### kun

Code Machine configuration and documentation (this repo).

- **URL**: [github.com/databayt/kun](https://github.com/databayt/kun)
- **Use For**: Claude Code team config, infrastructure docs

## Priority Order

When building features, check repos in this order:

1. **codebase** - Always check first for existing patterns
2. **shadcn** - For UI components
3. **radix** - For primitives
4. **Product repos** - For domain-specific patterns
