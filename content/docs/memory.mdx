# Memory

Memory files persist across conversations. They store component registries, project state, and references — so Claude remembers what was built and what patterns exist.

## Memory Files

| File | Contents |
|------|----------|
| **preferences.json** | Port, environment policy, package manager |
| **repositories.json** | Organization repos with paths + stacks |
| **atom.json** | Atom registry (composed primitives) |
| **template.json** | Template registry (full-page layouts) |
| **block.json** | Block registry (UI + business logic) |
| **report.json** | Report templates |

## Project-level Memory

Stored at `~/.claude/projects/<project>/memory/`:

| Slot | Contents |
|------|----------|
| Project profile | Active project status, stack, references |
| Active work | In-flight tasks, decisions, deadlines |
| User preferences | Feedback, corrections, working style |
| Capacity | Role assignments, availability |

## How Memory Works

Memory files are JSON or markdown in `~/.claude/memory/`. Claude reads them at the start of each conversation and updates them when new information is learned.

### Auto-memory

Claude automatically saves memories when it learns:
- **User info** — role, preferences, expertise level
- **Feedback** — corrections, confirmations, working style
- **Project state** — active work, decisions, deadlines
- **References** — where to find things in external systems

### Memory Index

`MEMORY.md` is the index file — always loaded into context. Each entry points to a memory file with a one-line description:

```markdown
# Memory Index

- [Project Profile](project_profile.md) — Active products and status
- [Capacity](capacity.md) — Role assignments
```

## Component Registry

The atom, template, and block memory files act as a registry across projects:

```json
{
  "atoms": [
    {
      "name": "tabs",
      "primitives": ["Tabs", "TabsList", "TabsTrigger", "TabsContent"],
      "path": "src/components/atom/tabs.tsx"
    }
  ]
}
```

When you say `atom tabs`, Claude checks the registry first before creating something new.

## Capacity Memory

`capacity.json` stores role assignments so agents can tailor allocations:

```json
{
  "roles": [
    {
      "id": "engineer-1",
      "role": "engineer",
      "focus": ["full-stack", "deployment"]
    }
  ]
}
```

Captain reads this on every weekly cycle to allocate work.
