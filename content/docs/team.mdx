# Roles

Four roles, each with a tailored MCP config, skill set, and agent index — so every team member has exactly the tools they need, nothing more.

## Roles

### Engineer
Engineering. Architecture, full-stack, deployment. Full agent fleet, all MCP servers.

### Business
Sales, outreach, proposals, customer communication. MCP set focused on github, slack, linear, stripe, notion, browser, context7.

### Content
Translation, documentation, research, UX writing. MCP set focused on github, slack, notion, browser, figma, context7, ref.

### Operations
Deploys, monitoring, incident response, cost tracking. MCP set focused on github, slack, vercel, sentry, neon, stripe, browser, posthog.

## Role-Specific MCP Configs

| Role | Config File | Servers |
|------|-------------|---------|
| **engineer** | `mcp.json` | All servers |
| **business** | `mcp-business.json` | github, slack, linear, stripe, notion, browser, context7, memory-bank |
| **content** | `mcp-content.json` | github, slack, notion, browser, figma, context7, memory-bank, ref |
| **ops** | `mcp-ops.json` | github, slack, vercel, sentry, neon, stripe, browser, posthog, memory-bank |

## Setup

```bash
# macOS / Linux
cd ~/kun && bash .claude/scripts/setup.sh engineer
cd ~/kun && bash .claude/scripts/setup.sh business
cd ~/kun && bash .claude/scripts/setup.sh content
cd ~/kun && bash .claude/scripts/setup.sh ops

# Windows (PowerShell)
cd ~/kun; .\.claude\scripts\setup.ps1 -Role engineer
```

### What Gets Installed

| Layer | Common (all roles) | Scoped (per role) |
|-------|--------------------|-------------------|
| **CLAUDE.md** | Shared instructions | — |
| **Agents** | All agent definitions | Role-specific index |
| **Skills** | docs, repos, screenshot, codebase | Role-specific skills |
| **Rules** | All rules | — |
| **Memory** | Shared registries | — |
| **Settings** | — | Hooks (engineer/ops) or minimal |
| **MCP** | — | Role-specific server set |

## Daily Patterns

### Engineer
```
> dev          # start dev server
> build        # validate
> push         # commit + push
> deploy       # ship
```

### Business
```
> /proposal "client name"
> /pricing
```

### Content
```
> /translate src/.../en.json
> /docs
```

### Operations
```
> /monitor
> /costs
> /incident <severity>
```

## Config Health

```bash
bash ~/.claude/scripts/health.sh
bash ~/.claude/scripts/health.sh --report
```

Or `/health` inside Claude Code:
- `/health` — local check + dashboard
- `/health report` — post to GitHub
- `/health all` — show all roles' latest status
